# AI 协作型人格图片区生图 Brief

## 0. 边界

这份文档只用于你在 ChatGPT image2 或其他外部生图工具里制作固定角色图，不是 skill 运行时提示词。

skill 运行时只会从 `assets/worktype-illustrations/manifest.json` 里选用已确认图片，再由确定性渲染器贴文字和分数；评分和出图时都不会再次调用生图模型。

目标资产是分享卡上方图片区。最终图片会放进 `996 x 666` 的横向图片区，底部会有白色淡出，所以不要按竖版全身海报构图。

## 1. 必守约束

- 正式比例：`166:111`，对应分享卡图片区 `996 x 666`
- 推荐尺寸：`1992 x 1332`
- 每张只有一个人类主角
- 同一批图保持同一画风、镜头距离、人物比例、材质、光照和复杂度
- 画风：克制的低多边形 Q 版 3D 角色，头部略大、身体短小、表情简化，几何切面清楚，哑光材质，柔和棚拍光，轻松但不幼稚
- 背景：干净暖色或低饱和纯色渐变背景，可用简单底座/平台/少量符号化道具，不做复杂办公室、科幻大厅或海报式大场景
- 色彩：每张一个主色氛围 + 一个识别 accent 色；允许不同人格换主色，但不要高饱和霓虹、彩虹渐变或蓝白科技风
- 禁止：文字、字母、数字、logo、水印、UI 假字、第二主角、密集海报背景、写实照片风、动漫风、幼儿贴纸风、过度卖萌、过度光滑的 3D 玩具质感
- 不复制 MBTI 或 16Personalities 的具体人物、服装、姿势、配色和构图

## 2. 构图原则

人物要在分享卡里看得清楚。脸、手、主动作、核心道具必须在画面中上部，不要放在底部或边缘。

建议构图：

- 人物为半身到七分身，不强求完整脚部
- 人物高度约占可见图片区的 `80%-90%`
- 人物宽度约占可见图片区的 `1/3`
- 镜头使用近景或中近景三分之二视角；可以略俯视，但不强制桌面视角
- 核心道具贴近人物手部，不能只是摆在背景里
- 底部只放底座、阴影或低优先级小物件
- 背景浅色、简洁、后退，不做远景建筑和复杂空间

一张图缩小到分享卡后，应先读到“这个人是谁”，再读到“他正在怎样和 AI 协作”。

## 3. 人格表达原则

每张图必须表达一种工作行为，不只是一个职业装扮。

合格图片要同时看出三件事：

- 人在做什么动作
- AI 或任务对象被他如何控制
- 任务状态正在发生什么变化

避免所有人格都变成“站在屏幕前看控制台”。每张图必须先有一个足够清晰、足够集中、足够有辨识度的主画面；互斥规则只是防跑偏，不是设计核心。

每张图只能服务一个“一眼画面句”：

- `赛博包工头`：站在工地符号前做整体指挥。
- `流水线督工`：盯住产线，把不合格输出拦下。
- `赛博对账人`：对照两组账卡，找出不一致。
- `边界判官`：一锤敲下，把问题边界切开。
- `抽象大师`：把散乱碎片捏成清晰模型。
- `落地狠人`：把散件压进交付包并扣上。
- `初稿仙人`：挥笔把空白变成第一版。
- `脑洞永动机`：把灯泡接线到下一个点子。
- `光速进度条`：双手推动进度条越过障碍。
- `智驾躺平哥`：看着 AI 自动跑，手边轻确认。

如果一张图需要解释两句话才知道它在干什么，说明人格表达不够集中，应减少道具和动作，而不是继续加元素。

## 4. 画面内容控制

每张图最多三个视觉焦点：

- 一个主角
- 一个核心道具体系
- 一个轻量任务对象或场景符号

推荐数量：

- 核心道具 `2-4` 个
- 小 AI 或任务符号块 `0-3` 个，优先做成小方块、小令牌、小连线，不要做成抢戏机器人
- 背景符号 `0-1` 组，尽量弱化

如果画面里每个角落都有东西，或小 AI 多到像群像，就算失败。清楚比热闹重要，道具要像证据、节点、按钮、箱子、进度块这类可一眼读懂的符号。

## 5. 通用生图提示词

每次生图先放这一段，再追加对应人格描述。

```md
Create one horizontal hero-slot illustration for an AI collaboration worktype share card. Canvas ratio 166:111, preferably 1992 x 1332. One central cute-proportioned workplace character in a restrained low-poly 3D style, slightly oversized head, compact body, simplified facial features, visible planar surfaces, chamfered edges, matte material, clean silhouette, warm soft studio lighting, simple low-saturation gradient background, close or medium-close three-quarter camera, playful but professional.

The image will be used only in the top illustration area of a share card, so keep the face, hands, main action, and identity-defining props clearly visible in the upper and middle area. Use one clear main action and one clear prop system. Prefer symbolic blocks, tokens, wires, tools, stamps, buttons, boxes, or progress pieces over large sci-fi scenery. No text, no letters, no numbers, no logos, no watermark, no fake UI text, no second main character, no dense poster background, no photorealism, no anime, no childish sticker style, no glossy plastic toy render.
```

## 6. 十种人格图表达

### 1. 赛博包工头 / 工作流架构型

核心人格：把混乱需求变成可施工的 AI 工作流。

职业锚点：工地总包 / 现场总指挥。

角色表达：Q 版小包工头，亮黄色安全帽略大，穿简化工程背心，表情稳、有主意。身体站得更开、更像在现场统筹，一手拿卷起的蓝图/对讲机，一手指向几个分区任务块，像在对全场下指令：“这边先做，那边等验收”。

画面重点：安全帽、对讲机或卷起蓝图、两三个分区任务块、小验收旗/工序桩。可以有一个小 AI 方块作为被指挥的辅助工，但不能出现一队机器人。

避免：传送带、跑偏输出盒、分流杆、质检夹板、复杂脚手架、控制台、满屏流程线。核心不是“亲自摆节点”，而是“站在现场做整体指挥”。

### 2. 流水线督工 / 输出调度型

核心人格：控制 AI 输出节奏，让结果按顺序、按规格过关。

职业锚点：监督员 / 产线质检。

角色表达：Q 版小督工，戴简化质检帽或袖章，姿态专注、略带“我盯着呢”的认真感。一手拿夹板/秒表/小哨子，另一手举起小拦停牌或指向一个不合格输出盒，像是在监督产线、发现问题、要求返工。

画面重点：短传送带、三四个输出盒、质检闸门/拦停牌、秒表或夹板。输出盒里有一个明显歪掉或停在检查点，监督动作必须一眼可读。

避免：安全帽蓝图、对讲机总指挥、施工分区、工程背心、复杂工厂、很多机械臂、进度条推进、训人姿态。核心不是“安排全局怎么干”，而是“盯住输出是否合格”。

### 3. 赛博对账人 / 证据校验型

核心人格：把两组依据放到一起对照，确认能对上再允许结论往前走。

职业锚点：对账审计员 / 双栏核验员。

角色表达：Q 版对账员，头稍低，眼神集中，拿着放大镜在两组并排令牌之间来回比对。情绪是冷静、细致、略微严肃，不要神秘侦探感。

画面重点：左右两列/上下两排对照令牌、一枚被放大镜圈出的不匹配令牌、连接两组的发光对照线。令牌可以用人物、文档、定位点、确认章等抽象图形表达，但不要出现文字。

避免：只有一组证据、单纯查看文件、法槌、边界线、迷宫、堆满文件、暗黑侦探、财务办公室。核心不是“裁定范围”或“破案”，而是“两边能不能对上”。

### 4. 边界判官 / 问题界定型

核心人格：先裁定问题边界，再开始解决。

职业锚点：边界裁判 / 范围法官。

角色表达：Q 版小判官，戴夸张但可爱的法官大波浪假发，表情坚定但不凶。举起小法槌，正在把混乱问题区域一锤定界。

画面重点：法官假发、法槌、一个被发光边界线切开的迷宫/问题块。边界线是主道具，问题块保持简洁。

避免：放大镜、证据令牌、审计桌、古装审判、巨大法院、攻击性表情、抽象云雾。核心不是“检查证据”，而是“问题先画清边界”。

### 5. 抽象大师 / 机制建构型

核心人格：把散乱现象压缩成可复用的结构模型。

职业锚点：机制建模师 / 结构塑形师。

角色表达：Q 版建模小大师，双手像揉面一样把几何碎片收拢成结构。表情专注，带一点“让我把它捏成模型”的笃定感。

画面重点：左侧少量碎片，中间双手塑形，右侧一个清晰结构球/模块塔。变化路径要清楚，但元素数量要少。

避免：灯泡、导线、点子爆炸、满屏概念泡、施工机器、只摆一个成品模型。核心不是“想法继续发散”，而是“混沌被压成结构”。

### 6. 落地狠人 / 交付封装型

核心人格：把半成品、检查项和结论封成可交付物。

职业锚点：交付封装员 / 发布收尾人。

角色表达：Q 版收尾高手，身体下压，双手把散件扣进一个小工具箱/发布包。表情利落，像在说“别聊了，封版”。

画面重点：工具箱或发布包、锁扣、两三个散件/检查令牌。动作是“压进去并扣上”，不是抱着箱子站着。

避免：进度条、障碍块、铁匠铺、仓库搬运、大量包裹、施工元素。核心不是“把进度往前推”，而是“把东西变成可交付物”。

### 7. 初稿仙人 / 初稿驱动型

核心人格：先把空白变成可修改的第一版。

职业锚点：快速起稿人 / 第一版生成者。

角色表达：Q 版起稿小仙人，动作轻快，一挥笔就把空白处铺出第一版。表情自信、灵动，但不要像魔法少女。

画面重点：空白画布/卷轴、三张飞出的草图卡或页面框、挥笔动作。草图卡要像“初稿”，不要像最终成品。

避免：灯泡、导线、概念泡、彩色烟花、满屏纸张、精修成品。核心不是“点子继续发散”，而是“先有一版”。

### 8. 脑洞永动机 / 概念发散型

核心人格：持续接出新想法、新分支、新方向。

职业锚点：灵感接线员 / 点子发动机。

角色表达：Q 版脑洞发动机，眼睛亮、姿态轻快，手里抓着一根导线，把发光灯泡接到下一个小概念泡上。感觉是“灵感还在续航”。

画面重点：发光灯泡、导线、两三个概念泡。概念泡数量少但方向感强，像在继续分叉。

避免：空白画布、页面框、草稿卡、满屏脑暴云、复杂星图、收束模型、坐着发呆。核心不是“产出第一版”，而是“一个点子接出下一个”。

### 9. 光速进度条 / 执行推进型

核心人格：目标明确后，把任务进度往前推。

职业锚点：进度推进员 / 任务清障员。

角色表达：Q 版推进小队长，身体明显前倾，双手推着一条发光进度条往前走。表情专注、有冲劲，但不是运动员冲刺。

画面重点：发光进度条、两三个被推开的障碍块、后方少量完成块。进度条要足够大，让“推动进度”一眼可读。

避免：发布包、锁扣、工具箱、复杂跑道、赛车、流水线分流、封箱交付。核心不是“收尾封装”，而是“把卡住的进度推过去”。

### 10. 智驾躺平哥 / AI主导型

核心人格：AI 先跑，人类在场观察并轻确认。

职业锚点：自动驾驶乘客 / 轻确认员。

角色表达：Q 版轻确认乘客，坐姿放松但眼睛还在看，手指靠近一个确认按钮。表情是省力、安心、在场，不是睡着或摆烂。

画面重点：自动驾驶椅或简化座舱、悬浮方向盘/自动运行符号、一个确认按钮。人物要轻松，但仍保留最低限度的验收动作。

避免：睡觉、躺倒、完全失控、人物废物化、满屏仪表盘。核心不是“什么都不管”，而是“AI 主跑，人类轻确认”。

## 7. 最终验收

每张图生成后只问五个问题：

1. 放进 `996 x 666` 图片区后，脸、手、主动作、核心道具是否清楚？
2. 不看文字，能否一句话说出这张图正在发生什么？
3. 画面是否只有一个主角、一个主动作、一个核心道具体系？
4. 是否没有文字、数字、logo、水印和 UI 假字？
5. 和其他九张图相比，动作、道具、职业锚点、轮廓是否足够不同？
